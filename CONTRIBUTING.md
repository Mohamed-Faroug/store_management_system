- اختبر التغييرات
- أرسل Pull Request

## 🛠️ إعداد بيئة التطوير

### 1. تحميل المشروع
```bash
git clone https://github.com/Mohamed-Faroug/store_management_system.git
cd store_management_system
```

### 2. إعداد البيئة الافتراضية
```bash
python -m venv venv
venv\Scripts\activate  # Windows
# أو
source venv/bin/activate  # Linux/Mac
```

### 3. تثبيت المكتبات
```bash
pip install -r requirements.txt
```

### 4. تشغيل التطبيق
```bash
python run.py
```

## 🚀 عملية الإرسال

### 1. إنشاء Fork
- اذهب إلى [المستودع](https://github.com/Mohamed-Faroug/store_management_system)
- اضغط "Fork"
- انسخ الرابط

### 2. تحميل Fork
```bash

cd store_management_system
```

### 3. إضافة Remote
```bash
git remote add upstream https://github.com/Mohamed-Faroug/store_management_system.git
```

### 4. إنشاء Branch
```bash
git checkout -b feature/your-feature-name
```

### 5. تطوير الميزة
- اكتب الكود
- اختبر التغييرات
- اكتب تعليقات

### 6. إرسال التغييرات
```bash

git commit -m "Add: وصف التغيير"
git push origin feature/your-feature-name
```

### 7. إنشاء Pull Request
- اذهب إلى GitHub
- اضغط "New Pull Request"
- اكتب وصفاً مفصلاً
- أرسل الطلب

## 🏷️ تسمية Branches

### أنواع Branches
- `feature/` - ميزات جديدة
- `bugfix/` - إصلاح الأخطاء
- `hotfix/` - إصلاحات عاجلة
- `docs/` - تحسين الوثائق
- `refactor/` - إعادة هيكلة الكود

### أمثلة
- `feature/user-management`
- `bugfix/login-error`
- `hotfix/security-patch`
- `docs/api-documentation`
- `refactor/database-queries`

## 📝 رسائل Commit

### تنسيق الرسائل
```
نوع: وصف مختصر

وصف مفصل للتغيير (اختياري)

- نقطة 1
- نقطة 2
- نقطة 3
```

### أنواع التغييرات
- `Add:` - إضافة ميزة جديدة
- `Fix:` - إصلاح خطأ
- `Update:` - تحديث ميزة موجودة
- `Remove:` - حذف ميزة
- `Refactor:` - إعادة هيكلة الكود
- `Docs:` - تحديث الوثائق
- `Style:` - تحسين التنسيق
- `Test:` - إضافة اختبارات

### أمثلة
```
Add: user authentication system

- Added login/logout functionality
- Added password hashing
- Added session management
- Added user roles

Fix: database connection error

- Fixed connection timeout issue
- Added error handling
- Improved connection pooling

Update: invoice generation

- Improved invoice layout
- Added new fields
- Enhanced printing options
```
